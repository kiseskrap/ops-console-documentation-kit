# Anonymization Checklist

Before publishing internal technical content:

- [ ] Remove company names.
- [ ] Remove real service names.
- [ ] Replace real domains with `example.com`.
- [ ] Replace API URLs with example paths.
- [ ] Remove bucket names.
- [ ] Remove distribution IDs.
- [ ] Remove API Gateway IDs.
- [ ] Remove Lambda function names.
- [ ] Remove database names and hosts.
- [ ] Remove cookie names and token names.
- [ ] Remove customer, partner, region, and organization names.
- [ ] Replace exact traffic/cost numbers with ranges when not essential.
- [ ] Replace production code with simplified examples.
- [ ] Generalize diagrams.
- [ ] Check screenshots for hidden identifiers.
