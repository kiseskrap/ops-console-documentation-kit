# Outputs

This repository produces reusable planning and documentation artifacts rather than a deployed application.

| Output | Purpose |
| --- | --- |
| `templates/HANDOVER_TEMPLATE.md` | System handover outline for new operators. |
| `templates/ADR_TEMPLATE.md` | Decision record format for why a technical choice was made. |
| `templates/TROUBLESHOOTING_TEMPLATE.md` | Incident investigation and recovery note template. |
| `templates/DEPLOYMENT_GUIDE_TEMPLATE.md` | Deployment runbook shape for repeatable releases. |
| `checklists/ANONYMIZATION_CHECKLIST.md` | Checklist before turning internal docs into public writing. |

## How to Use

1. Start from the README and pick the artifact that matches your current decision.
2. Copy the template or checklist into your own project.
3. Replace example names with your own system names.
4. Remove anything that exposes internal details before publishing or sharing.
